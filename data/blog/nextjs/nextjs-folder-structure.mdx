## Overall Architecture

```plain
src
├── app
│   ├── _components
│   ├── api
│   ├── favicon.ico
│   ├── globals.css
│   ├── layout.tsx
│   └── page.tsx
├── components
├── contexts
├── data
├── hooks
├── lib
├── services
├── types
└── utils
```

### app

Nextjs app router 사용시 기본적으로 존재하는 폴더이다. 

### api

app 폴더 내에 존재하는 api 폴더이다. Nextjs의 server component 도입으로 api route 기능을 대체할 수 있다는 말이 나오고 있지만 cross origin에서 데이터를 가져올 api가 필요할 때 유용하다.

### \_components

app router 방식에서 page.tsx와 layout.tsx에만 들어가는 컴포넌트 집합이다. 재사용 가능한 컴포넌트는 components 폴도로 이동시킨다. 상위 컴포넌트와 하위 컴포넌트의 관계를 폴더 구조로 사용한다.

### services

server action이나 data fetching api를 위한 폴더이다.

### components

재사용 가능한 컴포넌트들의 집합이다. 

### db

데이터 베이스와 직접 연결하는 경우 필요한 구성 파일들을 넣어둔다.

### types

재사용되는 타입을 이곳에 선언하고 관리한다.

### hooks

커스텀 훅을 모아둔 곳이다.

### public

이미지와 폰트를 넣어둔다. 이미지는 페이지나 목적에 따라 폴더를 구분한다.

### context

context를 저장하는 곳이다. context가 컴포넌트와 재사용되는 경우 이곳에 넣지 않고 해당 컴포넌트 안에 배치한다.

### data

사이트에서 사용하는 메타데이터나 정보들을 저장해두는 곳이다.

### lib

third-party library과 연관된 것들을 저장하는 곳이다. redux 같은 상태관리 라이브러리나 prisma 같은 orm도 포함된다.

### utils

여러가지 재사용되는 함수들을 보관하는 곳이다. 날짜 관련 함수나 tailwindcss merge 같은 함수들이 존재한다.
