## React는 왜 만들어 졌는가?

### 1. SPA (Single Page Application)

과거 웹 사이트는 매번 웹 서버로부터 페이지를 전달받아 브라우저에 표시되었다.
페이지를 불러오는 시간 지연은 사용자에게 좋지 못한 유저경험을 일으키고 페이지 정보가 커지면서 서버에 부담이 되었다.
이런 문제를 해결하기 위해 SPA 방식이 등장했다.
Single Page Application은 말그대로 단 하나의 페이지만 갖는 애플리케이션으로 하나의 페이지 안에서 JS를 통해 동적으로 페이지를 생성한다.

### 2. SPA의 문제점

SPA는 JS로 페이지를 동적으로 생성하여, 매번 서버에서 페이지를 받아오면서 발생하는 문제들을 해결했다.
하지만 애플리케이션 규모가 커지고 인터렉션이 많아지면서 JS로 DOM을 직접 조작하는 방식은 코드를 난잡하게 만들었다.
DOM을 업데이트하는 로직을 줄이고 많은 코드를 체계적으로 관리하기 위한 SPA 프레임 워크가 Angular, React, Vue 순으로 개발되었다.

### 3. 단방향 바인딩과 Virtual DOM

Angular는 프로퍼티 바인딩과 이벤트 바인딩이 결합된 양방향 바인딩을 지원했다.
양방향 바인딩은 상태의 변화에 따라 DOM을 어떻게 업데이트할지 규칙을 정해야 되기 때문에 애플리케이션 규모가 커짐에 따라 성능이 저하됐다.
React는 단방향 바인딩을 통해 데이터 흐름을 단순화하였다.
하지만 단방향 바인딩을 하게 되면 화면을 처음부터 다시 그려야했다.
React는 VirtualDOM을 통해 실제 DOM과 비교하여 업데이트가 필요한 컴포넌트만 렌더링하여 성능을 높였다.
